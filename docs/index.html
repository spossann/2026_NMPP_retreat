Struphy is …

… an open-source Python package for solving partial differential equations (PDEs), primarily designed for plasma physics — though not limited to it.

  • 💯% Python — Write models in clean, readable Python.
  • 💥 Pyccel-powered kernels — High-performance Fortran code generated via Pyccel.
  • 💻 Simple API — Run complex plasma simulations with minimal boilerplate.
  • 🌐 Open-source & community-driven — Built for collaboration and accessibility.

Numerical methods

Particle-in-cell (PIC) method for kinetic models:

\[ f(t, x, v) = \sum_k w_k\, \delta(x - x_k(t))\, \delta(v - v_k(t)) \]

Finite element exterior calculus (FEEC) for fluid/field variables (via Psydac):

\[ \mathbf E(t, x) = \sum_i e_i(t)\, \mathbf \Lambda^1_i(x)\,, \qquad \mathbf B(t, x) = \sum_i b_i(t)\, \mathbf \Lambda^2_i(x) \]

Smoothed Particle Hydrodynamics (SPH) for fluid models:

\[ n(t, x) = \sum_k w_k\, W(x - x_k(t))\,,\qquad \mathbf u(t, x) = \sum_k \frac{w_k}{n(t, x_k(t))}\, \mathbf v_k(t)\, W(x - x_k(t)) \]

Struphy provides MPI/OpenMP data structures for these methods (GPU via cuda-python in the second half of 2026 - ACH project Max Lindqvist)

Under consideration: Hermite polynomials for velocity discretization (C. Negulescu)

Purpose of Struphy

  • Provide a common Python API for researchers working on plasma physics PDEs

(such a thing exists for quantum physics, check out QuTiP).

  • Care about maintainable code, avoid “technical debt” in research software
  • Instead of losing research code, provide a framework for its preservation and reuse.
  • Enable computational plasma research for a wider audience, including students and researchers that are not experts in programming.

Installation Options

  1. Install on bare metal (Linux/macOS)
    1. From PyPI (recommended)
    2. From source (for development)
  2. Use Docker (cross-platform)
  3. Run tutorials on Binder (cloud-based)

🌐 Repo: github.com/struphy-hub/struphy

🌐 Docs: struphy-hub.github.io/struphy

🌐 Tutorials: github.com/struphy-hub/struphy-tutorials

Install on bare metal (Linux/macOS)

  • Use a virtual environment:
python -m venv struphy-env
source struphy-env/bin/activate
  • From PyPI (with physics packages):
pip install -U struphy[phys]
  • From source (for development):
git clone https://github.com/struphy-hub/struphy.git
cd struphy
pip install -e .[phys,dev]
  • Compile kernels (Fortran code generated via Pyccel):
struphy compile

Use Docker (cross-platform)

  • Pull the latest image from Docker Hub and run it:
docker pull spossann/struphy:main
docker run -it --rm spossann/struphy:main
  • List the existing environments and activate one:
ls struphy_*
source struphy_fortran_/env_fortran_/bin/activate
  • Check the compilation:
struphy compile --status

Run tutorials on Binder (cloud-based)

🌐 Tutorials: github.com/struphy-hub/struphy-tutorials

We will run through the first few tutorials together, but feel free to explore the rest on your own time!

Create a Simulation

from struphy import Simulation

model = Maxwell()
sim = Simulation(model=model, verbose=True)

Run

sim.run(verbose=True)

Time, grid, and Derham options

from struphy import Time, grids, DerhamOptions

time_opts = Time(dt=0.1, Tend=0.3)

grid = grids.TensorProductGrid(num_elements=(32, 1, 1))

derham_opts = DerhamOptions(degree=(3, 1, 1))

For more info on available options, see the API guide in the Struphy docs:

Create a new simulation object

The following simulation will inherit all the initial conditions and parameters from sim, except for the ones we explicitly override.

sim2 = Simulation(
    model=model,
    time_opts=time_opts,
    grid=grid,
    derham_opts=derham_opts,
    verbose=True,
)

You can compare sim objects

sim == sim2
False

We can serialze sims through dicts:

sim3 = Simulation.from_dict(sim.to_dict())

sim == sim3
True

This is the same as

sim4 = sim.spawn_sister()

sim == sim4
True

Only in devel so far - not yet in main:

sim.export("sim.yaml")
sim5 = Simulation.from_file("sim.yaml")
assert sim == sim5

Launch files - the API

Default launch files from CLI

struphy params MODEL

This will create a file params_MODEL.py in the current working directory.

To run the model type

python params_MODEL.py

For parallel runs, use

mpirun -n N python params_MODEL.py

or copy the command to a job script for your cluster’s scheduler (e.g., SLURM, PBS).

Conclusion

Plasma HPC framework in Python

  • Struphy 3.0: A structure-preserving hybrid code for plasma simulations in Python.
  • Combines particle and grid methods with high-performance kernels (Pyccel).
  • Easy-to-use API for setting up and running complex plasma simulations.
  • You can integrate your model into the API.
